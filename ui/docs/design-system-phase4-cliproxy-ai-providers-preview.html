Design Pattern — §1b Rail-anchored Identity

What changed

  • No top PageHeader — identity lives in left rail per design-system.md §1b
  • Left rail holds brand identity, primary CTA, provider family navigation
  • FamilyRail drives family selection via URL ?family= param (navigable, shareable)
  • JsonPane is editable (raw entry config) with settings.json preview alongside
  • Entry key prop forces remount on selection change (preserves original pattern)

Correctness preserved

  • preserveSecrets: don't send placeholder back to API when not rotating
  • effectiveSelectedEntryId: falls back to first entry when none selected
  • Bidirectional JSON sync: form → JSON (immediate), JSON → form (on valid parse)
  • 5-family dispatch: gemini, codex, claude, vertex, openai-compatibility
  • canSave: requires isRawJsonValid + no missing fields + no model errors + hasChanges
