Architecture Decision: PageHeader vs Rail-anchored

Codex is a single-entity config page (one global Codex CLI config, not a list). The decision table in design-system.md assigns:

  • Multi-entity Config → Rail-anchored (cliproxy pattern, Phase 2)
  • Single-entity Config → PageHeader (1c) + ConfigLayout + SectionRail

The SectionRail replaces the old Tabs — anchors map 1:1 to the existing tab labels (Overview / Control Center / Docs). RawConfigEditorPanel occupies the json slot — a deliberate choice since the existing panel is a full TOML editor, not a read-only JSON display.
