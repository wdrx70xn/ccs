Design System Insights

[Insight] Multi-entity Config pages with custom list sidebars (profile lists with promo cards, status badges, multi-action toolbars) do not map cleanly to the generic ListPane component — sub-component extraction is the right pattern rather than forcing the generic pane.
[Insight] Double-scroll lesson (PR #1093) applies here: ProfileEditor owns its internal scroll area, so FormPane wrapper is omitted. The pattern is documented in the file-level JSDoc comment.
[Insight] The code-editor usage contract test was updated to follow the bounded layout from api.tsx to api-profile-view-pane.tsx — contract tests must track sub-component extraction.
